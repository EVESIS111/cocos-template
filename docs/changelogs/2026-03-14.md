## Changed

### Game Performance - Bullet System (`Bullet.ts`)
- **Cached trigonometric calculations at initialization** instead of recalculating `sin`/`cos` every frame, reducing CPU cost in the bullet update loop. Since bullet angle is fixed at launch, this is a safe and effective optimization.
- **Reordered collision detection logic** to check boundary conditions (cheap) before bounding-box intersection tests (expensive), allowing early exit for out-of-bounds bullets without computing collisions.
- Introduced a shared `DEG_TO_RAD` constant to replace repeated inline degree-to-radian conversions.

### Game Performance - Main Loop (`Main.ts`)
- **Changed object pool retrieval from `shift()` to `pop()`**, improving bullet pool performance from O(n) to O(1) per retrieval. This directly reduces frame time spikes during heavy bullet usage.
- **Cached the `TheRectMask` component reference** at initialization, eliminating per-frame string-based `getComponent()` lookups.
- **Added position-change detection** for ball mask redraws, skipping unnecessary `drawRadius()` calls when the ball's Y position hasn't changed.

### Game Performance - Tracking System (`TrackFollow.ts`)
- **Eliminated per-frame `Vec2` object allocations** by computing delta values with raw arithmetic instead of creating temporary vector objects. This reduces garbage collection pressure during gameplay.
- Reuses a cached `Vec2` object for position updates via `setPosition()`.

### Game Performance - Vortex System (`VortexFollow.ts`)
- **Reads target position once per frame** instead of accessing it multiple times, reducing property lookup overhead.
- Reuses a cached `Vec2` object and a `DEG_TO_RAD` constant for cleaner, faster position calculations.

### UI - Rounded Mask Rendering (`TheRectMask.ts`)
- **Cached the `Graphics` component reference** and a reusable `Rect` object, eliminating repeated `getComponent()` calls and object allocations during mask drawing.

### UI - Ranking Leaderboard (`Rank.ts`)
- **Cached all child component references during initialization**, removing repeated `cc.find()` + `getComponent()` lookups from the `updateItem()` method. This is especially impactful for leaderboards with many entries.
- Replaced `forEach` iteration with a standard `for` loop for improved iteration performance.

### WeChat Integration - Video Ads (`wxcontrol.ts`)
- **Reuses the existing video ad instance** for the default ad unit ID instead of creating and destroying a new instance for every ad request. Custom ad IDs still create fresh instances as needed.
- Consolidated `videoLoading` state reset to prevent inconsistent loading states on error paths.

### Data Access - Local Storage (`Global.ts`, `utils/index.ts`, `Load.ts`)
- **Eliminated redundant `localStorage.getItem()` calls** across multiple modules. Previously, data was read twice per fetch (once to check existence, once to parse). Now reads once and reuses the result.

### WeChat Sub-Domain (`launch.js`)
- **Cached all leaderboard item component references** at initialization, matching the optimization pattern applied to `Rank.ts`.
- **Cached the SDK version string** from `wx.getSystemInfoSync()` to avoid repeated synchronous system calls.
- Replaced `forEach` loops with `for` loops for better performance.
- Cached the `none_data` Label component reference.
- Cleaned up commented-out code blocks.
