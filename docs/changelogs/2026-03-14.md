## Summary

This changelog covers performance optimization work across the game engine template. Two open pull requests ([#1](https://github.com/EVESIS111/cocos-template/pull/1) and [#2](https://github.com/EVESIS111/cocos-template/pull/2)) introduce significant runtime performance improvements by eliminating redundant computations, reducing memory allocations, and caching frequently accessed references. These changes target the game loop and UI rendering paths — areas where even small inefficiencies compound across thousands of frames.

**No breaking changes. No action required from users.**

## Changed

### Game Loop Performance (Bullet, TrackFollow, VortexFollow)

- **Cached trigonometric calculations in `Bullet.ts`** — `sin` and `cos` values are now computed once at bullet initialization instead of every frame, reducing per-frame math overhead for all active bullets. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

- **Optimized collision check ordering in `Bullet.ts`** — Boundary checks (simple coordinate comparisons) now run before the expensive `getBoundingBoxToWorld().intersects()` call, allowing most out-of-bounds bullets to exit early without triggering collision detection. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

- **Eliminated per-frame Vec2 allocations in `TrackFollow.ts`** — Replaced repeated `cc.v2()` object creation with a cached `Vec2` instance that is reused each frame, reducing garbage collection pressure during gameplay. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

- **Eliminated per-frame Vec2 allocations in `VortexFollow.ts`** — Reuses a cached `Vec2` for position updates and reads target position once per frame instead of twice. Added a `DEG_TO_RAD` constant to avoid repeated division. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

### Object Pool & Component Caching (Main.ts)

- **Changed bullet pool retrieval from `shift()` to `pop()`** — `pop()` is O(1) while `shift()` is O(n) due to array re-indexing. This improves bullet spawn performance, especially when the pool contains many recycled bullets. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

- **Cached `TheRectMask` component reference** — The mask component is now looked up once during `onLoad()` instead of every frame via string-based `getComponent('TheRectMask')`. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

- **Conditional mask redraw** — The round-rect mask is now only redrawn when the ball's Y position actually changes, skipping unnecessary draw calls when the ball is stationary. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

### UI Rendering (TheRectMask, Rank, launch.js)

- **Cached `Graphics` and `Rect` objects in `TheRectMask.ts`** — Avoids creating new `cc.Rect` objects and re-fetching the internal `Graphics` component on every `drawRadius()` call. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

- **Cached child component references in `Rank.ts`** — All `cc.find()` + `getComponent()` lookups for rank list items (rank label, cup sprite, head sprite, name label, score label) are now performed once during initialization and stored in an array. The `updateItem()` method uses these cached references instead of traversing the node tree on every page update. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

- **Cached child component references in `launch.js` (sub-domain)** — Same caching pattern applied to the WeChat sub-domain leaderboard rendering. Also replaced `forEach` with a `for` loop for better iteration performance, and cached the `none_data` Label component and SDK version string. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

### Data Access Optimizations (Global.ts, utils/index.ts, Load.ts)

- **Eliminated redundant `localStorage` reads in `Global.ts` and `utils/index.ts`** — `localStorage.getItem()` was being called twice per `fetchData()` invocation (once to check existence, once to read). Now reads once and reuses the result. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

- **Cached `fetchData()` result in `Load.ts`** — The loading scene was calling `Global.fetchData()` twice in sequence. Now calls once and stores the result in a local variable. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

### WeChat SDK (wxcontrol.ts, launch.js)

- **Reused video ad instance in `wxcontrol.ts`** — When using the default video ID, the existing `videoAd` instance is reused instead of creating and destroying a new one for each ad request. Custom video IDs still create fresh instances. This reduces WeChat SDK overhead and potential memory leaks. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))

- **Cached `wx.getSystemInfoSync()` result in `launch.js`** — The SDK version string is now fetched once and cached, avoiding repeated synchronous system info calls during version comparisons. ([PR #2](https://github.com/EVESIS111/cocos-template/pull/2))
